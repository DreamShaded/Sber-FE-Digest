## 🌎 Браузеры

- [Grid и flexbox научились рисовать разделители в зазорах без единого лишнего DOM-узла](https://developer.chrome.com/blog/gap-decorations-stable). Совместная работа команд Chrome и Edge: `column-rule` распространили на grid и flex, добавили `row-rule` для горизонтальных линий, а сверху навесили управление: где линия ломается на пересечениях, насколько отступает от краёв зазора, в каких зазорах вообще появляется.
- [HTML внутри `<canvas>` — предложение WICG, которое стирает границу между DOM и канвасом](https://tympanus.net/codrops/2026/05/13/exploring-the-html-in-canvas-proposal/). API состоит из трёх примитивов (`layoutsubtree`, `drawElementImage()` и событие `paint`) и рендерит настоящий HTML прямо в канвас, сохраняя вёрстку, CSS и доступность — то есть hero-секцию можно прогнать через шейдеры пост-обработки, а краулеры всё ещё видят живой DOM. Виттори показывает четыре сценария: пост-обработка, мелкие wow-эффекты вроде растворяющегося инпута, переходы между страницами и, самое сочное, полноценный 2D-интерфейс внутри 3D-сцены — в Three.js r184 уже появился `HTMLTexture` и `InteractionManager`, который через CSS matrix3d отдаёт hit-testing, hover и фокус браузеру нативно, без рейкастинга.

## 🌐 Веб-стандарты

- [В TC39 обсуждают новый примитив Composite для сравнения объектов по значению](https://frontendmasters.com/blog/arrays-objects-now-composites/). В JS два одинаковых по наполнению объекта, но не ссылающихся на один адрес в куче, при сравнении дают `false`, и использовать их как ключи Map тоже нельзя. Предложение Records & Tuples должно было это починить, но споткнулось о проблемы с производительностью. Composites — следующая попытка: `Composite.equal(pos1, pos2)` возвращает `true`, порядок ключей не важен, пропозал живёт и обсуждается, до релиза в спеку пока далеко.
- [Chrome и Edge тестируют новый HTML-элемент `<install>` для установки PWA без единой строки JavaScript](https://developer.chrome.com/blog/install-element-ot). `<install>` отдаёт браузеру контроль над кнопкой и не требует кода, однако может отличаться от желаемого дизайна, а `navigator.install()` — это промис-based JS API с полной свободой в дизайне UI, но без "встроенного доверия" к клику.

- [statewright: state-machine-гардрейлы для AI-агентов](https://github.com/statewright/statewright). Детерминированный движок на Rust (без LLM внутри) пускает агента только к тем инструментам, что разрешены на текущей фазе: в planning — read-only, edit открывается на implementing с лимитом строк, а деструктивные операции через Bash режутся, даже когда сам Bash разрешён. За время пилота агент проверил 3535 pull request'ов с 68% процентом закрытых проблем, а в основе архитектуры — два изолированных субагента: reviewer только читает и формирует структурированный отчёт, implementer исправляет код, и общаются они только через родительский оркестратор.
